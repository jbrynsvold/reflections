When would you want to use a remote repository rather than keeping all your work local?

When you are working with a team or when you want to work on your project and you don't have your home computer with you.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?

You may only want to update the remote repository every once in a while, because it is for bigger changes. The local one can have more small changes that people don't have to see.

Describe the differences between forks, clones, and branches. When would you use one instead of another?

Forks are done on github only. They create a copy of someone elses work, but always point back to the work.

Clones are taken from github to your computer or from a computer to another computer. They don't point back to the original.

Branches are done within a repository. 

What is the benefit of having a copy of the last known state of the remote stored locally?

So you can know the changes that have been made up until then, even if you haven't merged it yet. You can run a git diff to see those changes.

How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?

You would have to send files to each other or all work on the same file somehow, with Google Docs. But that would automatically save things when you aren't ready for that.

I suppose you could make a copy of a working file and do it manually, but this system makes it more streamlined.
